# 晓乐 (ST-Little-Player)

[English](./README.md) | 简体中文

SillyTavern 情景音乐播放器扩展。提供紧凑的、与 ST 主题深度融合的音乐播放组件，支持 AI 驱动的背景音乐选曲、歌词同步、斜杠命令控制。

## 功能

### 播放器组件

- **4 种显示模式**：吸附（贴合输入栏）、拖动（自由悬浮）、集成（嵌入输入区域）、隐藏
- **吸附对齐**：上/下、左/右
- **可折叠**：折叠时为迷你栏（封面 + 控制按钮），展开时为完整面板（封面 + 歌词 + 播放列表 + 搜索）
- **自定义透明度**：可选的背景透明度滑块（吸附/拖动模式）
- **毛玻璃效果 & 主题集成**：自动匹配你的 ST 主题
- **移动端支持**：响应式布局，触摸友好控件
- **键盘快捷键**：空格切换播放/暂停，Esc 收起面板

### 音乐来源

- **网络**：从在线音乐平台搜索并播放
- **服务端上传**：将音频文件上传至 ST 服务器播放
- **自定义 API**：配置自己的搜索和解析接口。详见下方[自定义 API 格式](#自定义-api-格式)。

可在 设置 -> 播放 中独立启用或禁用各来源。

### AI 选曲

AI 分析对话内容，自动选择适配场景的背景音乐。

**两种模式：**

- **提示词注入（Together）**：将 BGM 指令注入主 AI 提示词。AI 在回复中附带隐藏的歌曲推荐，系统自动解析并播放。无需额外 API，用你现有的 ST 连接即可。选曲成功后会弹出提示，歌曲会出现在聊天播放列表中。
- **函数调用（Function Call）**：注册隐式工具供 AI 调用，实现搜索、播放、停止音乐。需要官方 API 端点且支持函数/工具调用（如 OpenAI、Claude、Gemini、DeepSeek）。第三方转发/代理端点不一定支持工具调用。也可以配置一个独立的自定义端点。

**触发方式：**

- **自动触发**：新消息生成后自动分析场景并选曲
- **Greeting 触发**：加载角色卡首条消息时自动分析 BGM（需开启自动触发）
- **手动触发**：使用 `/xybgm` 随时触发分析（仅函数调用模式）。AI 会分析近期对话并选曲。

**其他：**

- **播放历史**：按聊天记录近期播放的歌曲，供 AI 参考
- **自定义提示词**：Together 模式支持编辑 BGM 指令提示词，可使用宏变量
- **聊天播放列表**：AI 选中的歌曲自动加入"聊天"播放列表标签页，可随时重播。刷新页面后清空。

### 斜杠命令

所有命令使用 `xy` 前缀：

| 命令 | 别名 | 参数 | 说明 |
|------|------|------|------|
| `/xyplay` | `/xyp` | `[歌曲名]` | 无参：切换播放/暂停；有参：搜索并立即播放 |
| `/xynext` | `/xyn` | - | 下一首 |
| `/xyprev` | `/xypp` | - | 上一首 |
| `/xyvol` | - | `[0-100]` | 无参：显示当前音量；有参：设置音量 |
| `/xyadd` | - | `<歌名> [歌手]` | 搜索并添加到播放列表（不立即播放） |
| `/xybgm` | - | - | 手动触发 AI 选曲分析（函数调用模式） |
| `/xynow` | - | - | 显示当前播放信息 |

**示例：**

```
/xyplay River Flows in You     # 搜索并播放指定歌曲
/xyplay                         # 切换播放/暂停
/xynext                         # 下一首
/xyvol 50                       # 设置音量为 50%
/xyadd Lemon                    # 添加歌曲到播放列表（不立即播放）
/xynow                          # 显示当前播放信息
```

### ST 宏

可在提示词预设、角色卡、STscript 中使用：

| 宏 | 说明 |
|------|------|
| `{{xiaoyueCurrentSong}}` | 当前播放的歌曲名 |
| `{{xiaoyueCurrentArtist}}` | 当前播放的歌手名 |
| `{{xiaoyueIsPlaying}}` | 正在播放返回 `true`，否则 `false` |
| `{{xiaoyueHasTrack}}` | 已加载曲目返回 `true`，否则 `false` |
| `{{xiaoyueRecentPlayed}}` | 近期播放的歌曲列表（最多 5 首，逗号分隔） |

**示例：**

```
当前BGM: {{xiaoyueCurrentSong}} - {{xiaoyueCurrentArtist}}
播放状态: {{xiaoyueIsPlaying}}
近期播放: {{xiaoyueRecentPlayed}}
```

### 歌词

- LRC 格式，时间同步显示
- 完整面板：滚动视图，带渐变遮罩
- 迷你栏：垂直滚动（吸附模式）或水平跑马灯（拖动模式）

### 国际化

- 中文和英文，根据 ST 语言设置自动检测

## 安装

### 通过 ST 扩展管理器安装（推荐）

1. 打开 SillyTavern -> 扩展 -> 安装扩展
2. 输入仓库地址：
   ```
   https://github.com/vvb7456/ST-little-player
   ```
3. 点击安装
4. 按提示重新加载页面

### 手动安装

1. 将仓库克隆到 ST 扩展目录：
   ```
   cd SillyTavern/data/default-user/extensions
   git clone https://github.com/vvb7456/ST-little-player.git
   ```
2. 重新加载 SillyTavern

## 使用

### 基础播放

1. 点击播放器组件展开面板
2. 点击控制栏左侧的 **搜索** 图标
3. 输入歌曲名，按回车搜索
4. 点击搜索结果立即播放，或点击 `+` 添加到播放列表
5. 点击 **播放列表** 图标查看队列，歌曲按来源分为标签页：
   - **网络**：从搜索添加的歌曲
   - **上传**：上传到服务器的文件（启用上传来源后显示）
   - **聊天**：AI 选中的歌曲（启用 AI 选曲后显示）

### AI 选曲

1. 进入 设置 -> AI
2. 开启 **AI 选曲** 开关
3. 选择模式：
   - **提示词注入**：主 AI 在回复中写入 BGM 推荐，无需额外 API。
   - **函数调用**：AI 调用工具搜索并播放音乐。需要支持工具调用的官方 API 端点，或配置自定义端点。
4. 开启 **自动触发** 可在新消息生成后自动分析
5. （可选）开启 **Greeting 触发**，打开角色卡时自动分析
6. 使用 `/xybgm` 可随时手动触发分析（仅函数调用模式）

**函数调用自定义端点设置**（需 OpenAI 兼容端点且支持工具调用）：

- **API 地址**：OpenAI 兼容端点（需支持 `/chat/completions` 和 `/models`）
- **API 密钥**：端点的 Bearer 令牌
- **模型**：点击连接按钮获取可用模型列表，或手动输入模型名
- **上下文消息数**：发送给 AI 的近期聊天消息数量（2-20，默认 8）

**歌曲找不到时：**

如果 AI 推荐的歌曲在所有已启用的音乐来源中都找不到，会弹出警告提示，当前播放不受影响。

## 设置

四标签页设置面板（外观 / 播放 / AI / 通用）：

- **外观**：播放器模式、吸附对齐、自定义透明度、拖动迷你文字开关
- **播放**：默认音量、播放模式（列表循环 / 随机 / 单曲循环）、音乐来源、播放列表导入/导出
- **AI**：AI 选曲开关、模式选择、自定义 API 配置、上下文消息数、自动触发、Greeting 触发、注入角色（系统/用户）、自定义提示词编辑器
- **通用**：设置导入/导出、关于

## 自定义 API 格式

自定义 API 音乐来源允许你配置自己的搜索和解析接口。

### 搜索接口

搜索 URL 中使用 `{keyword}` 作为占位符，扩展会将其替换为 URL 编码的搜索关键词。

**期望的响应**（JSON 数组）：

```json
[
  { "id": "123", "name": "歌曲名", "artist": "歌手", "duration": 240 },
  ...
]
```

### 解析接口

解析 URL 中使用 `{id}` 作为占位符，扩展会将其替换为 URL 编码的歌曲 ID（来自搜索结果）。

**期望的响应**（JSON 对象）：

```json
{
  "url": "https://example.com/audio.mp3",
  "name": "歌曲名",
  "artist": "歌手",
  "lyric": "[00:00.00] 歌词行...",
  "cover": "https://example.com/cover.jpg"
}
```

只有 `url` 是必需的。`lyric` 应为 LRC 格式。

## 数据与存储

| 数据 | 作用域 | 是否持久化 |
|------|--------|------------|
| 扩展设置 | 按用户 | 是 |
| 播放列表（网络 + 上传） | 按用户 | 是 |
| AI 选中的歌曲（聊天列表） | 按聊天 | 否（刷新后清空） |
| BGM 播放历史 | 按聊天 | 是 |
| 上传的音频文件 | 按用户 | 是 |
| 播放器状态（当前曲目、播放中） | - | 否（刷新后丢失） |

通过 ST 的"清理扩展数据"功能卸载时，将删除全部设置、播放列表和已上传的服务器文件。
